DB 스키마 + API 설계

PostgreSQL 테이블 구조와 FastAPI 엔드포인트

DB 테이블 (PostgreSQL)

📍 places
id UUID PRIMARY KEY
restaurant_name VARCHAR(200)
region VARCHAR(100) -- 성수동, 도톤보리
country CHAR(2) -- KR, JP, US
category VARCHAR(50) -- 카페/맛집/술집
address TEXT
lat DOUBLE
lng DOUBLE
insta_summary TEXT -- Claude 요약
instagram_url TEXT -- 원본 포스트
thumbnail_url TEXT
rating_info VARCHAR(100)
crawled_at TIMESTAMP
created_at TIMESTAMP
👤 users
id UUID PRIMARY KEY
firebase_uid VARCHAR UNIQUE -- Firebase 연결
email VARCHAR(200)
display_name VARCHAR(100)
created_at TIMESTAMP
🔖 bookmarks
id UUID PRIMARY KEY
user_id UUID → users.id
place_id UUID → places.id
created_at TIMESTAMP
UNIQUE(user_id, place_id)
📋 crawl_logs
id UUID PRIMARY KEY
started_at TIMESTAMP
finished_at TIMESTAMP
places_added INTEGER
status VARCHAR -- success/failed
error_message TEXT

API 엔드포인트 (FastAPI)

🌐 장소 조회 (공개)
GET /api/places/search
  ?region= &category= &country=
GET /api/places/trending
  최근 7일 새로 추가된 핫플
GET /api/places/{id}
🔑 북마크 (Firebase 토큰 필요)
GET   /api/bookmarks
  내 저장 장소 목록
POST /api/bookmarks/{place_id}
  장소 저장
DEL   /api/bookmarks/{place_id}
  저장 취소
👤 유저 (Firebase 토큰 필요)
POST /api/users/sync
  첫 로그인 시 DB에 유저 생성
⚙️ 파이프라인 (관리자 전용)
POST /api/admin/pipeline/run
  크롤링 수동 트리거
GET  /api/admin/pipeline/logs
  크롤링 기록 조회
