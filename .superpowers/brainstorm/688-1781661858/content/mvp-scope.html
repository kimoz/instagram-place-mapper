MVP 범위를 어떻게 나눌까요?

지금 당장 다 만들 수 있지만, 검증 속도와 출시 속도 사이의 균형이 중요합니다

현재 상황

✅ 백엔드 골격 있음  |  ✅ UI 프로토타입 있음  |  ❌ 실제 지도 없음 (Leaflet 미사용)  |  ❌ 실제 데이터 없음 (목업 5개)  |  ❌ 인스타 파이프라인 없음

1

Phase 1 — 핵심 검증 (4~6주)

  • React Native + Expo 앱 세팅
  • 진짜 지도 — react-native-maps + 실제 핀
  • 인스타 크롤링 파이프라인 — 해시태그 기반 수집 + Claude API로 요약
  • 지역 검색 + 장소 상세 페이지
  • PostgreSQL로 DB 교체 (SQLite 탈출)

→ 이것만으로도 "인스타 핫플 지도 앱"의 핵심 가치 검증 가능

2

Phase 2 — 리텐션 (+3~4주)

  • 피드 뷰 — 인스타 카드 스크롤 탐색
  • 카테고리 필터 — 카페/맛집/술집/기타
  • 국내/해외 탭 분리
  • 최근 본 장소 히스토리

→ 재방문율과 탐색 깊이를 높임

3

Phase 3 — 개인화 (+3~4주)

  • 북마크/저장 — 소셜 로그인 필요
  • 내 리스트 관리
  • 장소 공유 기능
  • 알림 — 내 동네 새 핫플 등록 시

→ 유저 계정이 필요해서 인증 시스템 추가됨

💡 제안: Phase 1만 먼저 완성해서 주변 10명에게 써보게 하는 게 제일 빠른 검증이에요.
"이 앱 또 쓰고 싶다"는 반응이 나오면 Phase 2+3 진행, 아니면 피벗.
